# Acciones

## Acciones del usuario

1. Confirmar el alcance del piloto:
   - Opcion recomendada: 10 preguntas de informatica + 10 preguntas de mantenimiento.
   - Alternativa: 20 preguntas de un solo banco.

2. Confirmar nombre visible de la app:
   - Propuesta: `OPE Trainer`.

3. Confirmar si en el piloto quieres incluir solo preguntas verificadas o tambien pendientes/errores:
   - Propuesta: incluir una mezcla controlada para probar bien la interfaz.

4. Probar la app en navegador cuando exista la primera version.

5. Probar instalacion en movil Android cuando la PWA este preparada.

## Acciones de Codex

1. Crear estructura del proyecto.

2. Definir modelo JSON.

3. Crear extractor Markdown a JSON para el piloto.

4. Crear app HTML/CSS/JS sin framework pesado.

5. Implementar:
   - modo estudio,
   - modo test,
   - filtros basicos,
   - estadisticas locales.

6. Implementar PWA:
   - `manifest.json`,
   - service worker,
   - cache offline,
   - iconos.

7. Validar con navegador local.

8. Preparar el proyecto para GitHub Pages.
